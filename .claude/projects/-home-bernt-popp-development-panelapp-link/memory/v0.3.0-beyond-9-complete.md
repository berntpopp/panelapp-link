The v0.3.0 plan (`docs/superpowers/plans/2026-06-17-panelapp-link-v0.3.0.md`)
was fully implemented via TDD on branch `beyond-9-v0.3.0` (16 commits on top of
the plan doc, as of 2026-06-17). Surface went 7 -> 9 tools.

What shipped: M1 fix (full-mode `n_*` mode-invariant), two aggregation tools
(`compare_panels`, `get_panels_for_genes` in `services/aggregations.py` +
`mcp/tools/aggregations.py`), additive `confidence_counts` on panel detail
(standard+), lean `minimal` `_meta` (drops upstream timing + `citation_short`),
opt-in OTLP tracing (`setup_tracing` + `otel` extra + `PANELAPP_LINK_OTEL__*`),
`PANELAPP_LINK_DATA__GENE_BATCH_CAP` (default 20), trimmed tool descriptions.

Final state: `make ci-local` green, coverage 95.3% (gate 85%),
`panelapp_service.py` byte-unchanged at 580 lines, all modules <=600.

**Plan gaps caught during execution (not in the plan, fixed):** several
existing 7-tool roster tests had to move to 9 (`test_app`, `test_tool_naming`,
`test_capabilities`, `test_tools`); the M1 full-branch needed a renamed local to
avoid strict-mypy `no-redef`; the `_Service` Protocol's `get_panel_genes` needed
`limit`/`offset` to structurally match the concrete service; the plan's
`setup_tracing` span test collided with the process-global TracerProvider already
set by `test_tracing.py` (dropped as redundant); `uv.lock`, README, and
`docs/usage.md` tool counts also needed updating. Not yet pushed / no PR opened.
